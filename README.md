## Проекты
<p> Проект 1: Калькулятор юнит-экономики онлайн-школы</p>
<p>Что нужно было сделать:<p>
<ol>
  <li>Компания готовится к масштабированию. Необходимо рассчитать юнит-экономику продукта и, учитывая цели на следующий год, полученные от руководства, составить сценарий развития.</li>
  <li>На основе полученного плана изменения основных метрик на следующий год и плана по привлечению студентов от отдела маркетинга составить план по маркетинговым костам.</li>
  <li>Подготовить план найма преподавателей.</li>
</ol>
<p>Как решала: На основе исторических данных были определены такие бизнес-метрики как средний CAC и средний CPA, Retention, LT и LTR, Средняя цена урока, переменные затраты – т.е. ЗП учителя на 1 урок, в качестве постоянных Фонд оплаты труда остальных сотрудников.
После был произведен расчет плановых показателей, куда была заложена возможность их процентного изменения руководством, при помощи формул создана связь между показателями и далее построен прогноз на 1 год вперед с учетом сезонности.
<p>Далее в расчет плановых показателей был заложен план по привлечению студентов в виде поправочного коэффициента на привлечение. По полученным данным создан дешборд по плану расходов для отдела маркетинга.
<p>План найма преподавателей был расчитан через функцию Поиск решения, учитывая ограничение по найму - не более 70 преподавателей в месяц.<p>

<p> Проект 2: Калькулятор юнит-экономики онлайн-кинотеатра</p>
<p>Что нужно было сделать:<p>
<ol>
  <li>Просчитать юнит-экономику продукта и предложить сценарий по настройке параметров для выхода на 25-ти процентную маржинальность.</li>
  <li>Собрать наглядную визуализацию, где будет показано, кто, где и в каком объеме смотрит фильмы на данной платформе.</li>
</ol>
<p>Имеющиеся данные: датасет по подписчикам (дата подписки и таймзона), датасет по просмотрам (время просмотра, id юзера и id фильма) и датасет по финансам по месяцам, включающий количество оплат, базовую цену, объем скидок, выручку, затраты на маркетинг и постоянные расходы.
<p>Как решала: Для расчета юнит экономики в качестве юнита была взята подписка. По исходным данным были рассчитаны средний Retention, Lifetime, LTR и CAC. На основе данных метрик определен показатель CAC на юнит, рассчитаны постоянные расходы на юнит. И на основе фактической цены была определена маржа на юнит как в абсолютном, так и в относительном выражении. Также было рассмотрено распределение подписчиков и просмотров в различных разрезах и подготовлена визуализация по полученным данным.<p>

<p>Выводы (итоги):<p>
<ol>
  <li>Создан калькулятор юнит-экономики, позволяющий оценить изменения конечных показателей эффективности, закладывая при этом различные изменения в базовых показателях, таких как цена, объем скидок, затраты на привлечение клиента. Был предложен сценарий выхода на 25-ти проценетную маржинальность за счет снижения маркетинговых затрат, поиска более дешевых и более эффективных каналов, а также небольшого увеличелия цены. </li>

</ol>

<p>Что нужно было сделать:<p>
<ol>
  <li>Провести анализ распределения количества покупок по виду (бесплатная/платная - флаг is_trial), по партнерам, а также по тому, какой по счету является покупка (первая, вторая и т.д.).</li>

<p>Проект 5: Моделирование изменения балансов студентов для онлайн-школы</p> 
<p>Что нужно было сделать:<p>
<ol>
  <li>Создать запрос, который собирает данные о балансах студентов за каждый "прожитый" ими день.</li>
  <li>Смоделировать изменение суммарного баланса. Создать визуализацию результата. </li>
</ol>
<p>Имеющиеся данные: датасет по всем транзакциям (оплатам и начислениям бонусов) по каждому студенту; датасет по всем занятиям студентов, включающий id студента, дату, статус и тип занятия<p>
<p>Как решала: Решение осуществлялось при помощи CTE конструкций. Сначала были определены все даты "жизни" студента после его первой транзакции. Далее найдены все изменения балансов студентов, связанные с успешными транзакциями, и определен кумулятивный баланс по каждому студенту. Также найдено изменение баланса из-за прохождения уроков и рассчитаны кумулятивные показатели (с отрицательными значяениями). Суммированием баланса по транзакциям и баланса по прохождению уроков найден искомый баланс каждого студента за каждый "прожитый" им день.

 <p>Выводы (итоги):<p>
